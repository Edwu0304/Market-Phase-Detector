# 維運手冊

## 1. 這個專案是怎麼來的

這個專案的起點，是這篇文章：

- [景氣循環投資的終極指南：地圖、時鐘與鏡子，三位大師的致勝策略](https://fintastic.trading/market_analysis/%E6%99%AF%E6%B0%A3%E5%BE%AA%E7%92%B0%E6%8A%95%E8%B3%87%E7%9A%84%E7%B5%82%E6%A5%B5%E6%8C%87%E5%8D%97%EF%BC%9A%E5%9C%B0%E5%9C%96%E3%80%81%E6%99%82%E9%90%98%E8%88%87%E9%8F%A1%E5%AD%90%EF%BC%8C%E4%B8%89/)

那篇文章把景氣循環投資拆成三個視角：

- 愛榭克：偏總體數據與景氣四階段
- 浦上邦雄：偏利率、資金與股市循環
- 霍華．馬克斯：偏心理、風險與信貸週期

這個專案把那個觀念落地成可持續更新的網站。做法不是複製文章，而是把概念轉成：

- 一套網站自己的總相位規則
- 三套作者獨立鏡頭
- 一套可以每月重跑、重新部署的固定流程

## 2. 系統定位

這不是常駐後端服務，也不是前端直接連資料源的網站。

正確理解方式是：

1. 用 Python 抓資料
2. 在本地或 CI 算出相位與鏡頭
3. 產出 JSON
4. 用靜態前端讀 JSON
5. 把整包網站部署到 Cloudflare Pages

所以這個專案的核心其實是資料整理與靜態輸出流程。

## 3. 目錄怎麼看

- `src/market_phase_detector/`
  核心 Python 程式
- `frontend/src/`
  靜態頁面、樣式、互動
- `data/`
  最新資料、歷史資料、頁面內容資料

- `tests/`
  測試
- `wrangler.toml`
  Cloudflare Pages 設定

## 4. 主要程式各自負責什麼

### `src/market_phase_detector/cli.py`

整個專案的總入口。

負責：

- 建立台美最新資料
- 建立台美共同月份歷史資料
- 建立三位作者鏡頭資料
- 輸出 `data/latest.json`
- 輸出 `data/history/*.json`
- 輸出 `data/site-content.json`
- 建立 `dist/`

### `src/market_phase_detector/live_pipeline.py`

把外部原始資料轉成專案要用的觀測值。

這裡定義了：

- 美國 FRED 序列代碼
- 台灣國發會資料網址
- 觀測值怎麼從原始資料轉出來

### `src/market_phase_detector/collectors/us_fred.py`

專門抓美國 FRED 資料。

### `src/market_phase_detector/collectors/tw_official.py`

專門抓台灣國發會 ZIP 資料，並解出本專案要的欄位。

### `src/market_phase_detector/engine/us_rules.py`

美國網站總相位候選規則。

### `src/market_phase_detector/engine/tw_rules.py`

台灣網站總相位候選規則。

### `src/market_phase_detector/engine/state_machine.py`

把單月候選值轉成比較穩定的正式相位，避免每個月亂跳。

### `src/market_phase_detector/lenses/metric_sets.py`

整理三位作者鏡頭共用的指標集合。

### `src/market_phase_detector/lenses/izaax.py`

愛榭克鏡頭規則。

### `src/market_phase_detector/lenses/urakami.py`

浦上邦雄鏡頭規則。

### `src/market_phase_detector/lenses/marks.py`

霍華．馬克斯鏡頭規則。

### `src/market_phase_detector/pipeline.py`

把網站總相位、三位作者鏡頭、歷史資料組成每個國家的最終快照。

### `src/market_phase_detector/exporters/json_exporter.py`

把最終資料寫成 `data/*.json`。

### `src/market_phase_detector/content.py`

網站共用文案。

### `src/market_phase_detector/strategy_content.py`

首頁與國別頁的理論、說明、策略文字。

### `src/market_phase_detector/site_builder.py`

把前端與資料組成 `dist/`。

### `frontend/src/app.js`

前端互動總入口，負責讀 JSON、畫首頁與國別頁內容。

### `frontend/src/styles.css`

站點樣式。

### `frontend/src/index.html`

首頁模板。

### `frontend/src/tw/index.html`

台灣頁模板。

### `frontend/src/us/index.html`

美國頁模板。

## 5. 所有指標與 API／官方資料來源

### 美國

美國資料全部來自 FRED。

專案目前用到的抓取方式：

- CSV：`https://fred.stlouisfed.org/graph/fredgraph.csv?id=<SERIES_ID>`
- JSON：`https://api.stlouisfed.org/fred/series/observations`

實際流程目前是用 CSV。

#### 美國指標對照

| 專案指標 | 資料序列 | 來源說明 | 官方頁面 |
|---|---|---|---|
| `leading_index_change` | `IPMAN` | 製造業工業生產月變化 | [IPMAN](https://fred.stlouisfed.org/series/IPMAN) |
| `claims_trend` | `ICSA` | 初領失業救濟金變化方向 | [ICSA](https://fred.stlouisfed.org/series/ICSA) |
| `sahm_rule` | `SAHMCURRENT` | 薩姆規則 | [SAHMCURRENT](https://fred.stlouisfed.org/series/SAHMCURRENT) |
| `yield_curve` | `T10Y2Y` | 10 年期減 2 年期殖利率差 | [T10Y2Y](https://fred.stlouisfed.org/series/T10Y2Y) |
| `hy_spread` | `BAMLH0A0HYM2` | 高收益債利差 | [BAMLH0A0HYM2](https://fred.stlouisfed.org/series/BAMLH0A0HYM2) |

#### 美國指標用途

- `leading_index_change`
  用於網站總相位，也會被三個作者鏡頭拿來當景氣方向代理。
- `claims_trend`
  用於網站總相位，也會被拿來看勞動市場壓力與風險升溫。
- `sahm_rule`
  主要用於衰退確認。
- `yield_curve`
  主要用於景氣轉折與資金條件判讀。
- `hy_spread`
  主要用於信用風險與風險偏好判讀。

### 如果你要換資料源

不要只改抓資料的網址。

至少要一起檢查這幾個地方：

1. `collectors/`
2. `live_pipeline.py`
3. `engine/`
4. `lenses/`
5. `tests/`
6. 這份維運手冊

原因很簡單：現在的規則、欄位名稱、前端顯示和測試，都是依照目前這批來源設計的。

## 6. 從抓資料到上 UI 的完整流程

### 高層流程

```text
外部資料源
  ├─ 美國 FRED
  └─ 台灣 國發會 ZIP
        ↓
collectors/*
        ↓
live_pipeline.py
        ↓
engine/* 算網站總相位
        ↓
lenses/* 算三位作者鏡頭
        ↓
pipeline.py 組國家快照
        ↓
json_exporter.py 寫 data/*.json
        ↓
site_builder.py 組 dist/
        ↓
Cloudflare Pages 部署 UI
```

### 細部步驟

1. 抓美國 FRED 資料
2. 抓台灣國發會 ZIP
3. 轉成台美最新觀測值
4. 轉成台美歷史觀測值
5. 各自先算網站總相位候選值
6. 用轉場規則確認正式網站總相位
7. 用三套鏡頭規則算出三位作者的獨立判讀
8. 組成國家快照
9. 寫出 `data/latest.json`
10. 寫出 `data/history/*.json`
11. 寫出 `data/site-content.json`
12. 把前端與資料組成 `dist/`
13. 部署到 Cloudflare Pages

## 7. 本地重建與檢查

```bash
pytest -q
python -m market_phase_detector.cli
```

如果要本地看頁面：

```bash
cd dist
python -m http.server 8765
```

然後打開：

- `http://localhost:8765/`
- `http://localhost:8765/tw/`
- `http://localhost:8765/us/`

## 8. 上線到 UI 的實際步驟

1. 確認測試通過
2. 跑 `python -m market_phase_detector.cli`
3. 確認 `data/latest.json`、`data/history`、`data/site-content.json` 有更新
4. 確認 `dist/` 已重建
5. 本地打開首頁、台灣頁、美國頁檢查內容
6. 執行部署命令

```bash
npx wrangler pages deploy dist --project-name market-phase-detector
```

## 9. 維運時常見檢查順序

### 如果頁面內容怪怪的

先看這個順序：

1. `data/latest.json`
2. `data/history/index.json`
3. `data/site-content.json`
4. `dist/`
5. Cloudflare Pages 線上頁面

### 如果手機和桌機內容不一致

優先檢查：

1. 是否部署的是同一個網址
2. `dist` 是否真的是最新重建版本
3. 前端是否有吃到舊快取

### 如果月份跳號或出現奇怪月份

優先檢查：

1. `data/history/index.json`
2. 各月份歷史檔
3. 共同月份過濾是否正確

再往下看：

4. `live_pipeline.py` 的月份對齊邏輯
5. `json_exporter.py` 的月份格式統一邏輯

### 如果畫面不對，但資料看起來對

優先檢查：

1. `dist/` 是否真的是最新重建版本
2. 你打開的是不是固定別名，而不是舊部署網址
3. 手機或瀏覽器是不是還停在舊快取

### 如果理論或說明文字不對

優先檢查：

1. `strategy_content.py`
2. `content.py`
3. `data/site-content.json`

## 10. 維運時不要做的事

- 不要讓前端直接連外部 API
- 不要把網站總相位誤寫成某位作者的結論
- 不要顯示台美無法對齊的假月份
- 不要只改前端文字卻不重建 `data/site-content.json`
- 不要直接手改 `data/history/index.json`，應該從程式重建
