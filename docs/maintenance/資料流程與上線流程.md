# 資料流程與上線流程

## 1. 一張圖先看懂

```text
外部資料源
  ├─ 美國 FRED
  └─ 台灣 國發會 ZIP
        ↓
collectors/*
        ↓
live_pipeline.py
        ↓
engine/* 算網站總相位
        ↓
lenses/* 算三個作者鏡頭
        ↓
pipeline.py 組國家快照
        ↓
json_exporter.py 寫 data/*.json
        ↓
site_builder.py 組 dist/
        ↓
Cloudflare Pages 部署 UI
```

## 2. 從抓資料到上 UI 的完整步驟

### 步驟 1：抓美國資料

程式：

- [us_fred.py](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/src/market_phase_detector/collectors/us_fred.py)
- [live_pipeline.py](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/src/market_phase_detector/live_pipeline.py)

做法：

1. 用 FRED CSV 端點抓指定序列
2. 解析成時間序列資料
3. 算出：
   - 月變動
   - 方向
   - 直接值

### 步驟 2：抓台灣資料

程式：

- [tw_official.py](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/src/market_phase_detector/collectors/tw_official.py)
- [live_pipeline.py](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/src/market_phase_detector/live_pipeline.py)

做法：

1. 下載國發會 ZIP
2. 解出 CSV
3. 抓出景氣燈號、領先、同時、失業、出口欄位
4. 算出本專案需要的方向與年增率

### 步驟 3：建立最新觀測值

程式：

- `build_us_observations`
- `build_tw_observations`

輸出概念：

- 美國一包最新觀測值
- 台灣一包最新觀測值

### 步驟 4：建立歷史觀測值

程式：

- `build_us_history_observations`
- `build_tw_history_observations`

輸出概念：

- 各自至少最近 24 個月左右的月資料
- 之後只保留台美共同月份

### 步驟 5：算網站總相位候選值

程式：

- [us_rules.py](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/src/market_phase_detector/engine/us_rules.py)
- [tw_rules.py](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/src/market_phase_detector/engine/tw_rules.py)

結果：

- 每個國家先得到一個候選相位
- 以及候選理由

### 步驟 6：做轉場確認

程式：

- [state_machine.py](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/src/market_phase_detector/engine/state_machine.py)

目的：

- 避免相位單月震盪
- 讓升級與降級要有確認條件

結果：

- 得到正式網站總相位
- 必要時附上 `watch`

### 步驟 7：算三個作者鏡頭

程式：

- [izaax.py](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/src/market_phase_detector/lenses/izaax.py)
- [urakami.py](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/src/market_phase_detector/lenses/urakami.py)
- [marks.py](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/src/market_phase_detector/lenses/marks.py)

做法：

- 每個鏡頭各自吃自己的代理指標
- 各自輸出：
  - 相位
  - 相位中文
  - 理由
  - 指標
  - 歷史列

### 步驟 8：組國家快照

程式：

- [pipeline.py](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/src/market_phase_detector/pipeline.py)

結果：

每個國家會變成一個完整物件，包含：

- `observations`
- `derived_signals`
- `decision`
- `handbook`
- `lenses`

### 步驟 9：寫出 JSON

程式：

- [json_exporter.py](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/src/market_phase_detector/exporters/json_exporter.py)

結果：

- `data/latest.json`
- `data/history/YYYY-MM.json`
- `data/history/index.json`
- `data/site-content.json`

這一步非常重要，因為前端只讀這些檔案。

### 步驟 10：建出可部署網站

程式：

- [site_builder.py](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/src/market_phase_detector/site_builder.py)

做法：

1. 清空舊 `dist/`
2. 複製 `frontend/src`
3. 把 `data` 複製進 `dist/data`

### 步驟 11：前端渲染 UI

程式：

- [app.js](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/frontend/src/app.js)

做法：

1. 首頁讀 `data/latest.json` 和 `data/site-content.json`
2. 台灣頁與美國頁也讀同兩份資料
3. 國別頁依 `country.lenses` 各自建立三個鏡頭區塊
4. 每個鏡頭用自己的 slider 控制自己的 `history`

## 3. 實際操作流程

### 本地更新資料

```bash
python -m market_phase_detector.cli
```

這個命令會一次完成：

- 抓資料
- 算相位
- 寫 JSON
- 重建 `dist`

### 本地驗證

```bash
pytest -q
python -m market_phase_detector.cli
```

如果要看畫面：

```bash
cd dist
python -m http.server 8765
```

然後打開：

- `http://127.0.0.1:8765/`
- `http://127.0.0.1:8765/tw/`
- `http://127.0.0.1:8765/us/`

## 4. 上線到 Cloudflare Pages 的步驟

### 第一次設定

1. 安裝依賴

```bash
pip install -e .
npm install
```

2. 登入 Cloudflare

```bash
npx wrangler login
```

3. 確認設定檔

設定檔位置：

- [wrangler.toml](/D:/repo/python/Market%20Phase%20Detector/.worktrees/strategy-map-redesign/wrangler.toml)

目前重點設定：

- 專案名稱：`market-phase-detector`
- 輸出目錄：`dist`

### 每次上線流程

1. 先跑測試

```bash
pytest -q
```

2. 重建資料與網站

```bash
python -m market_phase_detector.cli
```

3. 部署

```bash
npx wrangler pages deploy dist --project-name market-phase-detector
```

4. 打開部署網址檢查

至少檢查：

- 首頁是不是新版
- 台灣頁是否能看到三個鏡頭
- 美國頁是否能看到三個鏡頭
- 最新月份是否一致
- 沒有出現舊模板或舊卡片

## 5. GitHub Actions 或排程上線

如果之後要自動化部署，至少需要：

- `CLOUDFLARE_API_TOKEN`
- `CLOUDFLARE_ACCOUNT_ID`

建議自動流程順序：

1. 安裝 Python 與 Node 依賴
2. 跑 `pytest -q`
3. 跑 `python -m market_phase_detector.cli`
4. 跑 `npx wrangler pages deploy dist --project-name market-phase-detector`

## 6. 維運排錯順序

### 情況 A：畫面不對

先查：

1. `data/latest.json`
2. `data/history/index.json`
3. `dist/app.js`
4. 線上網址是不是固定別名而不是舊部署網址

### 情況 B：月份跳號或亂序

先查：

1. `live_pipeline.py` 的月份對齊邏輯
2. `json_exporter.py` 的 `_month_key`
3. `data/history/index.json`

### 情況 C：手機和桌機不一致

先查：

1. 你看的是否同一個網址
2. `dist` 是否有重建
3. HTML 是否有帶新版 `app.js` 與 `styles.css` 版本參數

### 情況 D：理論文案不對

先查：

1. `strategy_content.py`
2. `content.py`
3. `data/site-content.json`

## 7. 維運時不要犯的錯

- 不要只改 `frontend/src` 就直接部署，沒有重建 `dist` 一定會出問題
- 不要用頁面生成日期代替資料月份
- 不要把首頁網站總相位誤說成某位作者的結論
- 不要讓三個鏡頭共用同一個 slider 狀態
- 不要直接手改 `data/history/index.json`，應該從程式重建
