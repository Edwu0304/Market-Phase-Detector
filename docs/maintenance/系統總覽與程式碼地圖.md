# 系統總覽與程式碼地圖

## 1. 系統定位

這個專案是一個靜態網站型的市場週期儀表板。

它不是一個常駐後端服務，也不是前端直接連外部資料源。正確做法是：

1. 用 Python 抓台灣與美國資料
2. 在本地或 CI 先算出網站總相位與三個作者鏡頭
3. 產出 `data/latest.json`、`data/history/*.json`、`data/site-content.json`
4. 把 `frontend/src` 加上 `data` 複製成 `dist`
5. 把 `dist` 部署到 Cloudflare Pages

所以這個專案的核心其實是「資料整理與靜態輸出流程」，前端只是讀取整理好的 JSON。

## 2. 目錄結構

### 根目錄重要項目

- `src/market_phase_detector/`
  核心 Python 程式
- `frontend/src/`
  靜態前端頁面與互動邏輯
- `data/`
  產出的最新資料、歷史資料、站點文案資料
- `dist/`
  可直接部署的靜態網站輸出
- `tests/`
  測試
- `wrangler.toml`
  Cloudflare Pages 設定

## 3. 核心資料流

### 高層流程

1. `cli.py` 是總入口
2. `live_pipeline.py` 組合台美原始觀測值與歷史資料
3. `engine/` 先算網站總相位候選值與轉場確認
4. `lenses/` 再算三位作者各自的鏡頭判讀
5. `pipeline.py` 組裝成每個國家的快照
6. `exporters/json_exporter.py` 寫入 `data/`
7. `site_builder.py` 把前端與資料組成 `dist/`
8. Cloudflare Pages 讀 `dist/` 對外服務

## 4. 各程式碼介紹

### 4.1 `src/market_phase_detector/cli.py`

這是整個系統的總入口。

主要責任：

- 建立美國與台灣的最新資料
- 建立台美共同月份的歷史資料
- 建立三位作者的鏡頭資料
- 寫出 `data/latest.json`
- 寫出 `data/history/*.json`
- 寫出 `data/site-content.json`
- 產生 `dist/`

你可以把它理解成「資料建站總控台」。

### 4.2 `src/market_phase_detector/live_pipeline.py`

這個檔案負責把原始資料源轉成專案真正要用的觀測值。

主要工作：

- 定義美國用的 FRED 序列代碼
- 定義台灣國發會 ZIP 資料位址
- 把 FRED 原始序列轉成：
  - `leading_index_change`
  - `claims_trend`
  - `sahm_rule`
  - `yield_curve`
  - `hy_spread`
- 把國發會資料轉成：
  - `business_signal_score`
  - `leading_trend`
  - `coincident_trend`
  - `unemployment_trend`
  - `exports_yoy`
- 產出台美歷史月份資料

這個檔案是原始資料與內部判斷邏輯之間的轉換層。

### 4.3 `src/market_phase_detector/collectors/base.py`

提供最基本的 HTTP 抓取能力。

主要工作：

- GET JSON
- GET 文字

它本身不懂景氣資料，只負責基礎請求。

### 4.4 `src/market_phase_detector/collectors/us_fred.py`

美國資料收集器。

主要工作：

- 從 FRED 讀取資料
- 支援 JSON 與 CSV 兩種形式
- 解析成專案內部統一格式：
  - `rows`
  - `latest`

### 4.5 `src/market_phase_detector/collectors/tw_official.py`

台灣資料收集器。

主要工作：

- 從國發會下載 ZIP
- 解壓並讀取 CSV
- 抽出景氣燈號、領先指標、同時指標、失業、出口等欄位
- 組成可供後續判斷使用的月資料

### 4.6 `src/market_phase_detector/engine/us_rules.py`

美國網站總相位候選判斷。

目前依下列訊號決定候選相位：

- 領先指標變動
- 初領失業金方向
- Sahm Rule
- 10Y-2Y 殖利率差
- 高收益債利差

### 4.7 `src/market_phase_detector/engine/tw_rules.py`

台灣網站總相位候選判斷。

目前依下列訊號決定候選相位：

- 景氣對策信號分數
- 領先指標方向
- 同時指標方向
- 失業方向
- 出口年增率

### 4.8 `src/market_phase_detector/engine/state_machine.py`

轉場確認器。

用途不是直接判斷景氣，而是避免相位一個月就亂跳。

主要規則：

- 候選相位若與前期相同，直接採用
- 升級或降級通常要連續兩次候選都一致，才正式轉場
- 若是明確壓力情境，可用 `stress_override` 直接放行
- 否則會保留前期相位，並標記 `insufficient_confirmation`

### 4.9 `src/market_phase_detector/lenses/metric_sets.py`

三位作者鏡頭用的指標組定義。

它定義：

- 鏡頭名稱
- 每個鏡頭顯示哪些指標
- 指標標籤
- 顯示格式

### 4.10 `src/market_phase_detector/lenses/izaax.py`

愛榭克鏡頭。

目前用以下資料做代理：

- 領先指標變動
- 同時方向
- 勞動壓力
- 出口／生產方向

用途是用較偏總體與轉折的方式，看景氣四階段。

### 4.11 `src/market_phase_detector/lenses/urakami.py`

浦上邦雄鏡頭。

目前用以下資料做代理：

- 殖利率曲線
- 利率環境
- 市場領導廣度代理

用途是用利率結構與市場風格輪動的角度判斷週期。

### 4.12 `src/market_phase_detector/lenses/marks.py`

霍華．馬克斯鏡頭。

目前用以下資料做代理：

- 高收益債利差
- 估值／風險偏好代理
- 恐慌／自滿代理

用途是從信用、風險偏好與情緒面來看週期位置。

### 4.13 `src/market_phase_detector/models/phases.py`

定義網站總相位用的資料結構。

包含：

- `CandidateDecision`
- `TransitionDecision`
- `DecisionRecord`

### 4.14 `src/market_phase_detector/models/lenses.py`

定義鏡頭資料結構。

包含：

- `LensMetric`
- `LensDecision`
- `LensHistoryRow`
- `CountryLensBundle`

### 4.15 `src/market_phase_detector/pipeline.py`

把單一國家的資料組裝成標準快照。

輸出會包含：

- 國家代碼
- `as_of`
- `observations`
- `derived_signals`
- `decision`
- `handbook`
- `lenses`

### 4.16 `src/market_phase_detector/exporters/json_exporter.py`

負責把記憶體裡的結果寫成 JSON。

主要工作：

- 寫出 `data/latest.json`
- 寫出 `data/history/YYYY-MM.json`
- 重建 `data/history/index.json`
- 寫出 `data/site-content.json`
- 統一月份格式為 `YYYY-MM`
- 避免舊歷史月份殘留

### 4.17 `src/market_phase_detector/content.py`

負責組出站點內容總表。

輸出內容包含：

- 首頁文案
- 作者理論內容
- 國別分頁內容

### 4.18 `src/market_phase_detector/strategy_content.py`

這是站點顯示文案與理論說明來源。

包含：

- 相位中文名稱
- 國家中文名稱
- 三位作者的理論說明
- 各相位的定義、現象、判斷指標、操作重點
- 首頁方法說明與資料原則

### 4.19 `src/market_phase_detector/site_builder.py`

很單純，但很重要。

它做兩件事：

1. 刪除舊的 `dist/`
2. 把 `frontend/src` 與 `data` 複製進 `dist/`

也就是說，部署上線的內容完全以 `dist/` 為準。

### 4.20 `frontend/src/app.js`

前端主要邏輯。

主要工作：

- 判斷目前是首頁還是國別頁
- 讀取 `data/latest.json`
- 讀取 `data/site-content.json`
- 渲染首頁
- 渲染台灣頁與美國頁
- 管理每個鏡頭自己的月份拉桿
- 依拉桿位置更新表格、判讀與理論說明

首頁進入點。

### 4.23 `frontend/src/us/index.html`

美國頁進入點。

整個站的視覺與排版樣式。

網站現在這一刻要顯示的最新資料。

### 4.26 `data/history/*.json`

站點文案資料。

## 5. 最重要的維運觀念

### 5.1 前端不是資料真相

真正的真相來源順序是：

1. 外部資料源
2. `live_pipeline.py` 轉換後的觀測值
3. `engine/` 與 `lenses/` 的判斷結果
4. `data/*.json`
5. 前端畫面

如果畫面怪，先看 `data/*.json`，不要先怪 CSS。

### 5.2 `dist/` 才是部署真相

你改了 `frontend/src` 或 `src/market_phase_detector/*`，不代表線上就會變。

一定要重新跑：

```bash
python -m market_phase_detector.cli
```

這樣 `dist/` 才會重建。

### 5.3 歷史月份必須只看真實資料月份

不要再把 `generated_at` 當成歷史月份。

歷史月份應以每個國家的 `as_of` 對齊後決定，並由 `json_exporter.py` 統一格式與重建索引。
