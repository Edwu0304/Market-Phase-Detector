# Cloudflare Pages 空白頁教訓

日期：2026-04-09

## 這次真正出錯的地方

不是部署網址錯，也不是 Pages 沒發出去。

真正問題有兩層：

1. 前端入口檔與腳本快取混在一起，首頁一度還指向舊腳本。
2. 首頁入口 HTML 曾被寫壞，瀏覽器雖然拿到 200，但 DOM 解析不正常，最後呈現空白頁。

## 之後遇到空白頁，固定照這個順序查

1. 先確認正式站首頁 HTML
   - 檢查 `<script src="...">` 指向的是不是最新腳本檔名
   - 檢查 `<title>`、`<head>`、`<body>` 是否是正常完整 HTML

2. 再確認正式站腳本檔真的存在
   - 直接抓正式站的腳本 URL
   - 確認不是 404
   - 確認不是舊快取內容
   - 確認不是壞語法

3. 再確認資料檔
   - `data/latest.json`
   - `data/site-content.json`
   - 先確認能讀，再看欄位格式

4. 再確認瀏覽器執行期
   - 若 HTML、JS、JSON 都正常，才往執行期錯誤查
   - 優先看 `#app` 有沒有被渲染

## 禁止再犯

1. 不要只看 deployment 成功就認為網站可用
   - 一定要實際開正式站檢查畫面

2. 不要只給 preview / deployment URL
   - 先確認正式網址 `https://market-phase-detector.pages.dev/`

3. 不要沿用舊腳本檔名賭快取會更新
   - 有前端大改時，直接換新檔名最穩

4. 不要在沒確認首頁 HTML 正常前，就一頭栽進 JS 偵錯
   - 先看 HTML 結構，再看 JS

## 這次有效的修正方式

1. 改成新的腳本檔名，避免舊快取
2. 把首頁、台灣、美國三個入口 HTML 明確改成指向新腳本
3. 重新建置 `dist/`
4. 重新部署 Cloudflare Pages
5. 部署後直接抓正式站首頁 HTML 和腳本內容驗證

## 以後部署完成最低驗收

每次正式部署後，至少做這五件事：

1. 打開正式網址
2. 看首頁是否真的渲染，不是白屏
3. 看台灣頁是否可進入
4. 看美國頁是否可進入
5. 若有前端改動，確認正式站引用的是新腳本或新版本號
